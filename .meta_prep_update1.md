## THE SHIFT: What Meta Actually Tests

Meta's own prep doc says: "Don't worry about typos, syntax. If you aren't sure about syntax, just ask the interviewer."

In the AI era, they're testing: **does this person know what to build and why.** Not whether you can type it perfectly.

## PRIORITY ORDER (Non-negotiable)

| Priority | What | Safety Net? | Focus Level |
|----------|------|-------------|-------------|
| 1 | **Pattern recognition** — "this is a group-by-key problem" | NONE. Can't ask. | 60% of remaining time |
| 2 | **Logic flow** — "need dict, loop, check key, append" | NONE. Can't ask. | 30% of remaining time |
| 3 | **Syntax** — "is it .append() or .add()?" | YES. Can ask interviewer. | 10% — stop obsessing |

**You CAN ask:** "What's the syntax to check if an element is in a list?"
**You CANNOT ask:** "How should I approach this?" / "What pattern should I use?"

## WHAT WORKS (Proven)

| Method | Why |
|--------|-----|
| Treadmill trivia / pattern recognition quizzes | Tests Priority 1 without writing code |
| Step-by-step brick building (show → reproduce) | How Ariel actually locks patterns |
| Pseudocode → real code | Bridge between logic and syntax |
| Cold write next morning | Overnight consolidation is real (proved Day 5) |
| 2 sessions/day with break | Two sessions > one long grind |
| Stop when making new errors | Fatigue signal — stop, sleep, come back |

## WHAT DOESN'T WORK (Proven)

| Method | Why |
|--------|-----|
| Reading/watching without writing | Admiring ≠ owning |
| Claude Code for drill problems | Gives finished answers, kills the struggle that builds memory |
| One long grind session | Diminishing returns after 2-3 hours |
| Building tools instead of drilling | System creation as procrastination (documented pattern) |
| Syntax perfectionism | Safety net exists — don't waste time here |

## META INTERVIEW REMINDERS (From their official prep doc)

1. Think out loud — describe approach BEFORE coding
2. Get it correct first, optimize later
3. CAN ask for syntax help — it's explicitly allowed
4. Hit run to find bugs — don't manually check
5. Listen for hints — they WANT to help
6. Typos and syntax errors don't matter
7. Speed matters — practice with time constraints
8. No AI tools during interview — blank screen only

## THE BOTTOM LINE

Pattern recognition + logic flow = what gets you through the screen.
Syntax = learnable, askable, forgivable.
Volume on mediums = the one thing that builds pattern recognition.
Everything else is noise.
